# WEBDEV-PROJ-CROCHETE
This project is a dedicated study in design replication, built from the ground up to practice high-fidelity web development. To maintain a professional and scalable workflow, I have organized the source code into a modular structure hosted within this repository.

## 🏗️ Repository Organization
The project is split into specific branches/directories to keep the architecture clean and manageable:

HTML Structure: All core .html files are located within the html-files section. This serves as the structural foundation of the project.

Styling & Logic: All .css and .js files are housed within the css-and-javascript section. This ensures that the visual design and interactive behaviors are isolated from the markup.

## 🕒 Tracking Development History
You can monitor the evolution of this project by checking the Commit History. This allows you to see how the code has matured and what specific logic was implemented at different stages of the build.

To view the history:

1-Navigate to any file in the repo.

2-Select the "History" button in the top-right corner.

3-Review the individual commits to see the line-by-line changes.

### ⚠️ A Note on Commit Timestamps
Please note that the dates and times reflected in the GitHub history may not accurately represent the exact moment the code was written.
Because I often develop and refine the code within my local IDE throughout the day and push the final results in batches, there may be a delay between the actual coding sessions and the repository updates. The history is intended to show the logical progression of the code rather than a real-time log of activity
